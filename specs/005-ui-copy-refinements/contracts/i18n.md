# Contract: i18n Catalog

Scope: `src/i18n/en.ts`, `src/i18n/uk.ts`. The English catalog is authoritative; Ukrainian
mirrors every key. Lookup is `t(lang, key, params?)`.

## C-i18n-1: Two new keys, both catalogs

- `header.tagline` MUST exist in `en.ts` (under `// Header`) and `uk.ts` (under `// Header`).
- `summary.formula` MUST exist in `en.ts` (under `// Summary`) and `uk.ts` (under `// Summary`).
- Values MUST be the verbatim strings in `data-model.md` (exact punctuation: en-dash `–` in the
  weight range `1–3`, em-dash `—`, typographic apostrophe `’` in "don't" → `don't` uses ASCII `'`
  per the supplied EN copy; UA uses its supplied glyphs).

## C-i18n-2: Five relabeled display values

The values of `choice.empty`, `note.empty`, `form.addNote`, `form.noteTypeAria`, `form.textAria`
MUST be updated to the "new" column in `data-model.md`, in BOTH catalogs. Their KEY names MUST NOT
change.

## C-i18n-3: Untouched keys

`note.emptyShort`, `note.weightLabel`, `note.editAria`, all `noteType.*`, all `group.*`, all
`form.type*`, `form.textPlaceholder`, and every `choice.*` name/«Варіант» value MUST remain
exactly as they are today.

## C-i18n-4: Parity & integrity (auto-verified)

- `tests/unit/i18n.test.ts` parity test: `keys(en) === keys(uk)` (set equality). Adding both new
  keys to both catalogs keeps this green.
- "No blank / raw-key" test: every value is non-empty and `value !== key`. The new and relabeled
  values satisfy this.

## C-i18n-5: No identifier rename

No code identifier (`Note`, `NoteType`, function names, key strings) is renamed. This contract is
display-value-only plus two additive keys.
