# Implementation Plan: UI Polish — Toolbar Density, Collapsed-Card Summary & Contrast

**Branch**: `022-ui-polish-density-contrast` | **Date**: 2026-06-12 | **Spec**: [spec.md](spec.md)

## Summary

Five UX polish enhancements: toolbar `toolbar__field` density pattern with inline sub-options and
a "Create" container (US1, revised), empty-card footer tint carve-out (US3), WCAG AA compliance
for real labels (US4), three-voice CTA colour hierarchy with lightbulb micro-interaction (US5/US7),
and collapsed-header hover affordance (US6). US2 (collapsed card point summary) was implemented
then reverted after visual review. Save-status indicator extended with always-visible "Idle" state.
No schema change, no new runtime dependency, no store mutation change.

**Scale/Scope**: 6 user stories across 8 source files + 7 test files

## Implementation Phases

### Phase 1 — CSS foundation (US4 tokens + US5 classes)

Land the token and class changes before any component work, so downstream stories can reference
the new tokens confidently.

**Deliverables**:
1. `--warm` token in both `:root` and `[data-theme="dark"]`; `--status-editing: var(--warm)`.
2. `.btn--warm` + `.btn__bulb` filter rules in `app.css`.
3. `.btn--danger` in `app.css`.
4. `.toggle[aria-pressed="true"]` tint recipe replaces solid fill.
5. Selector changes: `.choice__scorelabel`, `.group-label`, `.actbtn`, `.saved` → `--text-muted`.
6. K1–K4 tests (class assertion in component tests) — TDD red first.

**Gates**: K1–K4 green; `tsc` clean; `yarn build` clean; existing suite green.
