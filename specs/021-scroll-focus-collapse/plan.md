# Implementation Plan: Scroll, Focus, Collapsible Header & Scrollbar Gutter

**Branch**: `021-scroll-focus-collapse` | **Date**: 2026-06-12 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/021-scroll-focus-collapse/spec.md`

## Summary

Four UX polish enhancements delivered as a single increment:

1. **Scrollbar gutter** (`app.css`) ‚Äî `scrollbar-gutter: stable` on `html`; eliminates layout shift when the scrollbar appears/disappears. CSS-only, zero logic.
2. **Auto-focus Add Point form** (`AddEditForm.svelte`) ‚Äî `use:autofocus` action on the textarea, extracted to a shared `src/actions.ts` module and imported by both `ChoiceCard.svelte` and `AddEditForm.svelte`.
3. **Auto-scroll to new Choice** (`App.svelte`) ‚Äî `$effect` watching `choices.length`; calls `tick()` then `scrollIntoView` on the last `.choice-cell` element; `smooth`/`instant` per `prefers-reduced-motion`.
4. **Collapsible header description** (`Header.svelte`) ‚Äî component-local `descOpen = $state(false)`; toggle button visible only at ‚â§719 px; description hidden by default on mobile, always visible on desktop; `slide` transition; two new i18n keys.

No new runtime dependencies. No `AppState` or `sift.v1` changes.
